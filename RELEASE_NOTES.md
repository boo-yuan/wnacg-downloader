# WNACG Downloader v1.0.0 Release Notes

🎉 经过漫长的深度打磨与反复重构，**WNACG Downloader** 终于迎来了它的首个正式工业级版本 v1.0.0！
这不仅仅是一个下载器，而是一个兼具**极端容灾能力、类人反爬隐蔽性、以及现代 Fluent Design 美学**的全自动化引擎。

---

## 🚀 核心亮点 (Key Features)

### 1. 全局令牌桶限流 (Global Token Bucket Pacer)
告别了传统爬虫粗暴的“并发齐射”。底层重构引入了全局 asyncio.Lock() 与 Semaphore，**所有并发请求会排队通过一个极窄的时间阀门**。配合内置的 Jitter（0.7x~1.3x）随机抖动，你的每次请求都会像真正的人类一样自然错开，彻底瓦解 Cloudflare 等高级 WAF 的瞬间流量检测。

### 2. 深度防撕裂校验 (Deep Image Validation)
抛弃了仅靠文件头的弱校验。现在下载器在每一张图片落地后，都会强制通过 PIL.Image.load() 试探其文件流末尾。任何因为网络闪断导致截断的“半身图”、“灰底图”，都会被立刻识别并打回重下，**确保你画廊里的每一张老婆都是 100% 完整的**。

### 3. 智能多轨域名容灾 (Multi-Domain Fallback Routing)
永远不用担心 wnacg.ru 突然被墙或 DNS 污染。系统内置了动态备用域名池（.org, .net, .com 等）。当某个请求遭遇连接超时，核心引擎会无感、静默地切换到下一个存活节点并继续重试，保障下载流如丝般顺滑。

### 4. 彻底告别被墙的“一键热更新” (One-Click Proxy Auto-Updater)
内置热更新系统。当你点击“检查更新”时，它不再直接连接 GitHub，而是**智能走国内极速公益加速镜像源（如 kkgithub）**。这确保了大陆用户也可以在无需代理的情况下，一键满速拉取最新版本。

### 5. 极致的性能压榨与数据一致性
- **SQLite WAL 机制**：解锁了读写死锁的枷锁，即使多协程高频写入进度，UI 层的数据查询依然畅通无阻，毫秒级响应。
- **TLS 连接池复用**：curl_cffi 从局部抛弃，进化为单例共享，彻底砍掉了昂贵的握手开销，性能暴增。

## 🛠️ 其他改动 (Changelog)
- **UI**：将“单任务并发数”优化整合为“全局最大并发连接数”，更符合网络连接的底层逻辑。
- **UI**：支持 Shift 连选、Ctrl+A 全选、鼠标框选，并在顶部加入了醒目的交互提示。
- **CI/CD**：新增 .github/workflows/build.yml，现在只要推送 Tag，GitHub Actions 会自动编译构建出极其纯净的无黑框 .exe 单文件。

> **安装提示**：解压即用。如遇更新报错，请直接使用设置中的“检查更新”重新拉取最新版本即可！
